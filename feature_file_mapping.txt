 FEATURE 1 — SECURE LOGIN / AUTHENTICATION
================================================================================
================================================================================

  PURPOSE
    Handles user registration, login with password hashing, and logout.
    Validates phone (must start with 98, 10 digits), email format, and
    password strength (min 8 chars, 1 uppercase, 1 digit, 1 special char).
    Uses werkzeug check_password_hash / generate_password_hash.

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  BACKEND                                                                │
  └─────────────────────────────────────────────────────────────────────────┘
    app/controllers/auth_controller.py
      ├── AuthController.login()            POST: verifies hashed password
      ├── AuthController.register()         POST: validates & creates user
      ├── AuthController.logout()           clears session, redirects
      ├── AuthController.validate_phone()   regex: must start with 98, 10 digits
      ├── AuthController.validate_password()min 8 chars, uppercase, digit, symbol
      └── AuthController.get_current_user() reads session → returns user dict

    app/security.py
      ├── Security.hash_password()          SHA-256 hashing (legacy utility)
      ├── Security.verify_password()        compares hashed passwords
      └── Security.generate_wallet_id()     returns "EP" + random hex token

    app/auth.py                             login-required decorators / guards
    app/session_manager.py                  sets session permanent_lifetime
    config.py                              SECRET_KEY for session signing

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  FRONTEND                                                               │
  └─────────────────────────────────────────────────────────────────────────┘
    HTML Templates:
      app/templates/login.html              login form + flash message display
      app/templates/register.html           registration form with all fields
      app/templates/base.html               master layout (nav, flash alerts)

    CSS Stylesheets:
      app/static/css/login.css              login & register page layout/styles
      app/static/css/style.css              global base styles
      app/static/css/theme.css              color tokens, CSS custom properties

    JavaScript:
      app/static/js/login.js                client-side login form validation
      app/static/js/register.js             client-side register validation
      app/static/js/style.js                global UI helpers, flash auto-dismiss

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  DATABASE                                                               │
  └─────────────────────────────────────────────────────────────────────────┘
    register          Stores full user profile + hashed password
      columns: username, password, full_name, email, phone,
               epaisa_id, balance, address, account_type, date_joined

    login             Active session snapshot (created on login, deleted on logout)
      columns: username, password, full_name, epaisa_id

    administration    Admin accounts only
      columns: id, admin_email, password

    NOTE: Password hashed with werkzeug on register. security.py SHA-256
          is a secondary/legacy utility. Both exist in the codebase.

 FEATURE 2 — PASSWORD RESET (FORGOT PASSWORD)
================================================================================
================================================================================

  PURPOSE
    User submits phone → 6-digit OTP generated (valid 5 min) → OTP verified
    → user sets new password. OTP currently held in in-memory dict (otp_store).

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  BACKEND                                                                │
  └─────────────────────────────────────────────────────────────────────────┘
    app/controllers/password_controller.py
      ├── PasswordController.request_reset(phone)
      │     generates 6-digit OTP, stores in otp_store{} dict with 5-min expiry
      └── PasswordController.verify_reset(phone, otp)
            checks otp_store for matching code and valid expiry

    app/error_handler.py → ValidationError   raised on bad input

    NOTE: OTP is stored in an in-memory Python dict (otp_store) — data is lost
          on server restart. A persistent `password_resets` DB table is
          the recommended improvement.
          Also: query uses "SELECT * FROM users WHERE phone=?" but actual
          table is `register` — needs alignment.

 FEATURE 3 — RECEIVE MONEY (SEND / TRANSFER)
================================================================================
================================================================================

  PURPOSE
    User enters receiver wallet ID and amount → PENDING transaction created →
    OTP issued → on OTP success, receiver balance updated, status = SUCCESS,
    and a notification is pushed to receiver.

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  BACKEND                                                                │
  └─────────────────────────────────────────────────────────────────────────┘
    app/controllers/transaction_controller.py
      ├── TransactionController.create(sender_wallet, receiver_wallet, amount)
      │     validates receiver exists & is ACTIVE
      │     inserts PENDING transaction (fee hardcoded Rs.5)
      └── TransactionController.complete(transaction_id)
            updates receiver wallet balance (balance + amount)
            sets transaction status = SUCCESS
            calls NotificationController.create()

    app/controllers/notification_controller.py
      └── NotificationController.create(wallet_id, title, message)
            inserts notification record after successful transfer

    app/controllers/otp_controller.py
      ├── OTPController.generate(transaction_id)  creates OTP for transfer
      └── OTPController.verify(transaction_id, otp)

    app/error_handler.py → WalletError           raised on wallet validation fail

    NOTE: Sender balance deduction must be handled separately in routes.py
          or wallet_controller.py — not inside TransactionController.create().
          Transaction fee is hardcoded as TRANSACTION_FEE = 5.

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  FRONTEND                                                               │
  └─────────────────────────────────────────────────────────────────────────┘
    HTML Templates:
      app/templates/wallet/wallet.html          send money form (enter receiver + amount)
      app/templates/wallet/verify_otp.html      OTP confirmation step
      app/templates/wallet/result.html          transaction result summary
      app/templates/wallet/success.html         "Transaction Successful" screen
      app/templates/wallet/failed.html          "Transaction Failed" screen

    CSS Stylesheets:
      app/static/css/wallet.css                 wallet & transfer page styles
      app/static/css/style.css
      app/static/css/theme.css

    JavaScript:
      app/static/js/wallet.js                   transfer form logic, OTP input
      app/static/js/style.js                    global helpers

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  DATABASE                                                               │
  └─────────────────────────────────────────────────────────────────────────┘
    wallets           sender checked for ACTIVE status, receiver balance updated
      columns: wallet_id, user_id, balance, status

    transactions      new record created as PENDING, updated to SUCCESS/FAILED
      columns: id, sender_wallet, receiver_wallet, amount, fee, status, created_at

    otp_verifications OTP generated and stored per transaction
      columns: id, transaction_id, otp_code, expires_at

    notifications     notification pushed to receiver on SUCCESS
      columns: id, wallet_id, title, message, created_at

 FEATURE 5 — OTP FLOW (TRANSACTION OTP)
================================================================================
================================================================================

  PURPOSE
    A 6-digit OTP is generated per transaction with a 5-minute expiry window.
    Used before finalizing money transfers. Separate OTP also used for
    password reset (in-memory store). Auth OTP stored in DB.

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  BACKEND                                                                │
  └─────────────────────────────────────────────────────────────────────────┘
    app/controllers/otp_controller.py
      ├── OTPController.generate(transaction_id)
      │     random.randint(100000, 999999), stores in otp_verifications
      │     expiry = datetime.now() + timedelta(minutes=5)
      └── OTPController.verify(transaction_id, otp)
            fetches latest OTP for transaction (ORDER BY id DESC LIMIT 1)
            checks otp_code match and expiry

    NOTE: No resend / invalidate mechanism. Only the latest OTP per
          transaction_id is checked. Expired OTPs remain in DB uncleaned.

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  FRONTEND                                                               │
  └─────────────────────────────────────────────────────────────────────────┘
    HTML Templates:

      app/templates/auth/verify_otp.html        auth / password-reset OTP page

    CSS Stylesheets:
      app/static/css/wallet.css
      app/static/css/style.css
      app/static/css/theme.css

    JavaScript:
      app/static/js/wallet.js                   OTP digit input, auto-submit
      app/static/js/style.js

  ┌─────────────────────────────────────────────────────────────────────────┐
